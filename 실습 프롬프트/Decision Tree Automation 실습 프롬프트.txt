[Decision Tree Automation 실습 프롬프트]

다음 과제를 Decision Tree 기반 의사결정 자동화 방식으로 분석해줘.

주제: ‘AI 문서 자동화 시스템에서 발생하는 문제를 자동으로 분류하여 처리 주체를 결정한다.’

아래 단계를 순서대로 수행해줘.

1) 의사결정이 필요한 상황 정의

문제가 발생했을 때 어떤 선택지를 결정해야 하는지 3줄로 요약해줘.
예: 개발팀/운영팀/보안팀 중 누구에게 배정할지 판단

2) 의사결정 기준(Key Decision Criteria) 5개 도출

예: 문제 유형, 심각도, 사용자 영향도, 보안 관련 여부, 데이터 손상 여부 등
각 기준이 왜 중요한지도 1줄씩 설명해줘.

3) Decision Tree 분기 조건 설계

위 기준을 기반으로 아래 형식으로 분기 조건을 만들어줘.

형식 예시:
If {조건}: → {담당팀}
Else if {조건}: → {담당팀}

분기 조건은 최소 5개 이상 구성해줘.

4) 전체 Decision Tree를 트리 구조로 시각화

아래와 같은 형태로 계층구조를 만들어줘.

예시:

문제 발생
 ├─ 데이터 손상 여부?
 │    ├─ Yes → 보안팀
 │    └─ No → 다음 단계
 ├─ 심각도 High?
 │    ├─ Yes → 개발팀
 │    └─ No → 운영팀

5) 자동화된 최종 처리 규칙(Automation Rules) 작성

각 팀에 배정되는 조건을 1줄 요약으로 정리해줘.

예:
개발팀: 시스템 오류, 심각도 높음
운영팀: 사용법 문의, 경미한 오류
보안팀: 데이터 무결성 문제 발생 시

6) 실무 적용 방안 제안 (3가지)

이 Decision Tree를 실제 자동화 시스템에 적용할 수 있는 방법을 제시해줘.
예: 워크플로우 자동화, 알림 시스템, API 기반 라우팅 등

출력 형식:
상황 정의
의사결정 기준
분기 조건
Decision Tree 구조
자동화 규칙
적용 방안
